国土交通省が日本の中央省庁として初めて MCP サーバーを公開しました。これにより、道路・橋梁・河川・災害・都市計画など **302万件以上** のデータに、Claude などの AI との対話でアクセスできるようになります。この記事では、具体的なプロンプト例を添えて活用アイデアを45個紹介します。後半には、"使い方のコツ" と "セットアップ方法" を掲載しています。

[» 国土交通データプラットフォーム](https://www.mlit-data.jp/#/Page?id=apps_mcp)

## MCP サーバーとは？

MCP（Model Context Protocol）は、AIアプリケーションと外部サービスを「USB-C のように」つなぐ共通規格です。従来は API の仕様を理解し、クエリを書き、レスポンスを解析する必要がありましたが、MCPサーバー経由なら、自然言語で話しかけるだけでデータを取得できます。

## 国土交通データプラットフォームで使えるデータ

- 道路・橋梁・トンネルの点検データ
- 交通量調査データ
- PLATEAU（3D都市モデル）
- 河川・ダム・水資源データ
- 洪水浸水想定区域・土砂災害警戒区域
- 港湾・空港インフラデータ
- 地質・ボーリングデータ
- 旅客動態・観光データ
- 都市計画・用途地域データ

## プロンプトのサンプル (使い方の事例)

### インフラ・建設業界向け

#### 1. 橋梁点検優先度分析

`○○県内で劣化度Ⅲ以上かつ交通量1万台/日以上の橋梁リストを出して、交通量の多い順に並べて`

#### 2. 道路補修計画の立案支援

`国道○号線沿いの舗装データで、劣化が進んでいる区間を抽出して、補修優先度をつけて`

#### 3. 工事現場周辺の地質調査

`○○市△△地区の半径500m以内にある地質・ボーリングデータを全部見せて`

#### 4. トンネル維持管理データ収集

`○○県内のトンネルで、点検結果が「要注意」以上のものを築年数順にリストアップして`

### 交通・物流向け

#### 5. 交通量調査レポート作成

`東京都庁付近で最も交通量の多い道路を10件教えて。時間帯別の傾向も含めて`

#### 6. 物流拠点最適化分析

`○○港周辺の道路交通量データと港湾施設データを組み合わせて、物流拠点としての特性を分析して`

#### 7. 渋滞ポイント特定

`首都高速○○線沿いの交通量データを取得して、ボトルネックになっていそうな区間を特定して`

#### 8. 旅客動態分析

`○○県の主要観光地への旅客動態データを取得して、季節ごとの傾向を分析して`

### 都市計画・不動産向け

#### 9. 都市開発ポテンシャル評価

`○○駅から徒歩10分圏内の都市計画データ、用途地域、人口動態を取得して、開発ポテンシャルを評価して`

#### 10. PLATEAU 3D都市モデル活用

`○○市のPLATEAU（3D都市モデル）データの一覧を見せて。建物LOD2データがあるエリアを教えて`

#### 11. 空き地・低未利用地調査

`○○区内の都市計画データから、開発余地のありそうなエリアを分析して`

#### 12. 駅前再開発検討資料

`○○駅周辺500m以内の道路データ、交通量、都市施設データを取得して再開発の検討資料にまとめて`

### 防災・危機管理向け

#### 13. ハザードマップ統合分析

`○○市の洪水浸水想定区域と土砂災害警戒区域のデータを重ね合わせて、複合リスクの高いエリアを特定して`

#### 14. 避難所アクセシビリティ評価

`新宿駅近くの避難所を10件取得して、それぞれのアクセスルートと収容人数を教えて`

#### 15. 河川氾濫リスク分析

`多摩川流域の河川水位データと堤防データを取得して、氾濫リスクの高い地点を特定して`

#### 16. 災害時インフラ脆弱性評価

#### 17. BCPルート検討

### 地方自治体・行政向け

## 使い方のコツ

### 位置指定は自然言語でOK

`札幌テレビ塔の近くにどんなデータがあるか教えて` のように、住所や施設名で指定すると、MCP が自動的に緯度経度に変換してくれます。

### あいまい検索にも対応

`災害に関するデータ` のような曖昧なクエリでも、適切なデータを探してくれます。

### 条件の追加・絞り込み

一度結果を見てから `さらに○○で絞り込んで` と追加条件を指定できます。

### ダウンロードURL取得

`ファイルのURLを教えて` と言えばダウンロードリンクが取得できます。

## セットアップ方法

1. [国土交通データプラットフォーム](https://www.mlit-data.jp/) でアカウント作成
2. API キーを取得
3. [GitHub リポジトリ](https://github.com/MLIT-DATA-PLATFORM/mlit-dpf-mcp) をクローン
4. Claude Desktop の設定ファイルに MCP サーバーを登録
5. Claude Desktop を再起動

詳細な手順は[公式の説明書](https://www.mlit-data.jp/cms//api/v1/pages/_page/apps_mcp/MCP_InstallationGuide.pdf)が非常にわかりやすいので、そちらを参照してください。

## まとめ

国土交通省の MCP サーバー公開は、日本のオープンデータ活用において大きな一歩です。

これまで「データにたどり着けない」という問題が、建設DXや地域分析のボトルネックになっていました。MCP サーバーにより、API の知識がなくても誰でも自然言語でデータにアクセスできるようになります。

今後、e-Stat など他のオープンデータプラットフォームにも MCP 対応が広がれば、AI × オープンデータの可能性はさらに拡大していくでしょう。
