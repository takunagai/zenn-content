Subagent が登場した 2025年7月から現在（2026年3月）までに Claude Code に追加された機能・コマンドを時系列で一覧にまとめました。

:::message
情報は 2026年3月29日時点のものです。バージョン番号・リリース日は公式 CHANGELOG および各種コミュニティソースをもとに整理していますが、一部に誤りがある可能性があります。正確な情報は公式ドキュメントをご確認ください。
:::

## 2025年7月 ── Subagent 登場

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|
| Custom Subagents | v2.0.x | `.claude/agents/` に Markdown を置くだけで専用コンテキスト・ツール制限付きエージェントを定義。Task ツールで起動 | フロント/バックエンド分業、ドメイン特化エージェント |
| Plan Mode / Plan Subagent | v2.0.x | Shift+Tab×2 でプラン専用モードに移行。実行前に変更内容を確認・承認。Plan Mode 専用の内蔵調査 Subagent が提案を生成 | アーキテクチャ設計時の事前調査・破壊的変更の安全確認 |
| Custom Slash Commands | v2.0.x | `.claude/commands/` に Markdown ファイルを置くとカスタムスラッシュコマンドとして登録。`$ARGUMENTS[N]` で引数渡し可能 | プロジェクト固有の繰り返し操作のコマンド化 |

## 2025年8〜9月 ── Checkpoints・VS Code 拡張

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|
| Checkpoints / `/rewind` | v2.0.x | プロンプトごとに自動チェックポイントを生成（30日保持）。Esc×2 または `/rewind` でロールバック。Chat only / Code only / Both の3モードで選択的復元 | 長時間自律タスクの安全実行・ミス時の即時復元 |
| VS Code 拡張 (beta) | v2.0 | ネイティブ VS Code 拡張を正式リリース。インライン diff・サイドバーパネル・セッション継続。Cursor / Windsurf とも連携 | ターミナル不要の IDE 内完結ワークフロー |
| `/statusline` | v2.0.x | ターミナルのプロンプトバーにカスタムステータス表示を追加。シェルスクリプトで model・tokens・git branch 等を自由にフォーマット | 使用状況・モデル・コストのリアルタイム可視化 |

## 2025年10月 ── Subagent 非同期・Hooks 拡張

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|
| Subagent 再開・非同期化 | v2.0.28 | 完了 Subagent のコンテキスト保持再開・バックグラウンド継続が可能に。SubagentStart / SubagentStop フック追加 | 長時間タスクの中断再開・並列バックグラウンド処理 |
| PermissionRequest Hook | v2.0.28 | カスタムロジックでツールの許可・拒否を自動判定するフックイベント。hooks に PermissionRequest を追加することで承認フローを自動化 | CI/CD での自動承認ポリシー実装 |
| Background Tasks (Ctrl+B) | v2.0.28 | Ctrl+B で実行中のコマンドをバックグラウンドに移行。メインセッションをブロックしない。`&` プレフィックスでも同様 | dev サーバー起動・長時間ビルドの並行実行 |

## 2025年11月 ── Agent Skills・Opus 4.5

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|

| `.claude/rules/` サポート | v2.0.x | CLAUDE.md に加え、ルールファイルを分割管理できる専用ディレクトリが追加。ファイルパスマッチ等の条件付きルールも記述可能 | プロジェクト別・ロール別ガイドラインの構造化管理 |

## 2025年12月 ── Remote Control・/loop・Named Sessions

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|
| `/loop` | v2.0.51 | 定期実行・監視ループ。`/loop 5m check the deploy` のように間隔とプロンプトを指定。Cron スケジューリングツールも同時追加 | デプロイ監視・テスト自動化・定期レポート生成 |
| Remote Control / `/rc` | v2.0.60 | claude.ai/code 経由でブラウザ・スマートフォンからセッション継続・タスク投入。`&` プレフィックスで Web UI からバックグラウンドタスク送信も可能 | 外出中・スマホからのタスク投入・進捗確認 |
| `/rename`・Named Sessions | v2.0.64 | `/rename` でセッションに名前付け。`/resume <名前>` で名前指定再開が可能に。resume 画面で P プレビュー・R リネーム | 複数プロジェクト並行管理・セッション切り替え効率化 |
| `/voice` | v2.0.x | 音声入力対応。Push-to-talk（スペースキー長押し）で話しかけてコードを操作。20言語対応（英・日・仏・独・西・露・中・韓 等） | ハンズフリーでのコーディング指示 |

## 2026年1月 ── v2.1.0 メジャーアップ・Skills 強化

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|
| v2.1.0（1096 コミット） | v2.1.0 | Hooks のスコープ化（スキル・エージェント・コマンド単位）。Skills ホットリロード。タスク管理システム追加 | 複雑なエージェントワークフローの精度・再現性向上 |
| `/keybindings` | v2.1.x | `~/.claude/keybindings.json` をエディタで開き、キーバインドをカスタマイズ。コンテキスト別・コード列にも対応。変更即時反映 | 個人ワークフロー最適化 |
| `/doctor` | v2.1.x | Claude Code の環境診断を実行。依存関係・設定・認証状態を一括チェック | セットアップ問題の切り分け |
| Instant Compact（Auto-compaction） | v2.1.16 | コンテキスト上限に近づくと自動でサマリ圧縮。`/compact [保持したい内容]` で手動実行・保持内容の指定も可能 | 長時間セッションの延命・大規模リファクタリング |
| `/branch`（旧 `/fork`） | v2.1.x | 現在の会話をブランチして別アプローチを安全に実験。元の会話は保持。v2.1.77 で `/fork` から `/branch` にリネーム（`/fork` はエイリアスとして継続） | リファクタリング案の比較・危険な変更前の保険 |

## 2026年2月 ── /batch・/simplify・Agent Teams・Opus 4.6・Auto Memory

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|
| Agent Teams ⚠️実験的 | v2.1.32 | 複数セッションが直接通信して協調作業。team lead + teammates 構成。`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` で有効化。Opus 4.6 必須 | 並列開発・大規模リファクタリング・マルチ視点コードレビュー |
| `/copy` | v2.1.59 | コードブロックをインタラクティブに選択してクリップボードにコピー。`/copy N` で N番目の回答を指定。`w` キーでファイル直接書き出し（SSH 環境向け） | SSH 越しでのコードブロック抽出 |
| Auto Memory | v2.1.59 | Claude が作業中に有用な文脈を自動でメモリファイルに保存。`/memory` で管理。`autoMemoryDirectory` 設定で保存先変更可能 | セッションをまたぐプロジェクト知識の自動蓄積 |
| `/simplify` | v2.1.63 | 複雑なコード・説明を3エージェント並列で自動整理するバンドル済みスキル。アーキテクチャ問題・重複ロジック・パフォーマンス非効率を検出 | PR 前のコード品質チェック・ドキュメント簡略化 |
| `/batch` | v2.1.63 | 複数ファイルへの同一変換を worktree 分離で安全に並列実行、自動 PR 作成。コードベースを調査→5〜30 ユニットに分解→バックグラウンドエージェントで並走 | 大規模コード移行・統一フォーマット化・API 変更の一括適用 |
| HTTP Hooks | v2.1.63 | フック処理を URL への JSON POST で外部サーバーに委譲する新しいフックタイプ。`"type": "http"` を hooks 設定に追加するだけ | Slack 通知・外部ログ・承認 API との統合 |
| Worktree 設定共有 | v2.1.63 | 同一リポジトリの git worktree 間でプロジェクト設定・Auto Memory を共有。v2.1.72 でワークツリーサブエージェント分離も追加 | 並列ブランチ開発での設定一貫性確保 |
| Opus 4.6 デフォルト化 / ultrathink | v2.1.68 | Opus 4.6 がデフォルトモデルに。Max/Team はデフォルト effort が medium に変更。`ultrathink` キーワードで次のターンだけ high effort に昇格。Opus 4 / 4.1 は廃止 | コスト最適化しつつ重要な場面で最高精度を引き出す |

## 2026年3月 ── /btw・/effort・MCP Elicitation・Native Plugins・1M Context

| 機能・コマンド | バージョン | 概要 | 実務での使いどころ |
|---|---|---|---|
| `/btw` | v2.1.72 | タスク実行中にサイドチャンネルで質問できるバンドル済みスキル。回答はオーバーレイ表示で会話履歴に残らない。Claude が処理中でも独立して実行可能 | コンテキスト汚染なしの確認・トークン消費削減（最大50%節約との報告も） |
| `/context` 強化 | v2.1.72 | コンテキスト逼迫の原因特定と最適化提案を表示。重いツール・メモリ肥大・残量警告を特定し具体的な改善ヒントを提示 | 長時間セッションのコンテキスト効率化・ボトルネック特定 |
| Native Plugins（公式マーケットプレイス） | v2.1.72 | 公式プラグインマーケットプレイス。`/plugin install` でインストール・`/plugin validate` で検証。スキル・エージェント・コマンド・Hooks をパッケージ化 | カスタムツール一元管理・チーム・組織への配布 |
| `/effort` | v2.1.72 | モデルの推論努力レベルを low / medium / high で動的に変更。`/effort auto` でリセット。スキル・エージェントの frontmatter でも指定可能 | コスト重視 vs 精度重視の柔軟な切り替え |
| `/color` | v2.1.75 | セッションのプロンプトバーに色を設定してセッションを視覚的に識別 | 複数セッション並走時のセッション識別 |
| Opus 4.6 × 1M Context | v2.1.75 | Max / Team / Enterprise プランで Opus 4.6 の 1M トークンコンテキストがデフォルト有効。出力上限も 128K トークンに拡大（v2.1.77） | 超大規模コードベースの全体把握・長文ドキュメント処理 |
| MCP Elicitation | v2.1.76 | MCP サーバーがタスク途中で構造化入力を要求できる双方向ダイアログ。Elicitation / ElicitationResult フックでインターセプト可能 | 自律タスク中断なしに追加情報が必要な場面での動的入力収集 |
| claude.ai MCP Connectors 統合 | v2.1.69 | claude.ai で設定した MCP サーバーが Claude Code でも自動で利用可能に。`ENABLE_CLAUDEAI_MCP_SERVERS=false` でオプトアウト可能 | claude.ai と Claude Code の MCP 設定を一元管理 |
| InstructionsLoaded Hook | v2.1.69 | CLAUDE.md または `.claude/rules/*.md` がロードされた時点で発火するフックイベント追加 | CLAUDE.md 適用の監査ログ・条件付き初期化処理 |
| StopFailure Hook | v2.1.78 | API エラー（レート制限・認証失敗等）でターンが終了した際に発火するフックイベント | エラー発生時の自動リカバリーフロー実装 |
| Channels ⚠️実験的 | v2.1.80 | MCP サーバーからセッションへのプッシュ型メッセージ送信機能。`--channels` フラグで有効化 | 外部イベント駆動の自律エージェントトリガー |
| `--bare` フラグ | v2.1.81 | スクリプト用 `-p` 呼び出しで Hooks・LSP・プラグイン同期をスキップし軽量起動。`ANTHROPIC_API_KEY` が必須 | GitHub Actions 等での高速バッチ CI 処理 |
